# Main / Child Agent

Child Agent 是运行时执行方式，不是 AIPD 流程里的必经阶段。Case / Work Package 承接目标和状态；Main / Child 决定当前工作由谁完成。

## Main Agent 保留什么

Main 负责用户沟通、意图与边界判断、主线设计、调度、结果审核、验收和文件状态写回。

以下任务通常适合 Main 连续完成：

- 单一路径或内聚模块。
- 实现、调试和前序判断高度耦合。
- 上下文规模可控。
- 派发、等待和合并成本高于节省的成本。

“调研”“多文件修改”“测试”这些标签本身都不构成派发理由。

## Child Agent 什么时候有净收益

### 上下文隔离

长文档、长日志、大量页面结构、批量扫描等过程会产生高噪声，而 Main 最终只需要少量结论。

### 真实并发

存在两条以上独立、可同时推进、不会写入冲突的工作线，墙钟时间收益明确。

### 独立复核

高风险结论需要第二视角，而且复核者不必重复继承整条主线。

派发的收益要覆盖启动、上下文继承、协调、等待、合并和重复工作的成本。

## Child 的最小上下文

默认传递文件入口、边界和返回格式，而不是复制完整聊天：

- Case 和当前 Work Package 路径。
- 选定的领域角色或指引。
- 允许修改 / 不允许修改的范围。
- 验收标准和预期返回。

Child 自己读取这些文件校准任务，只回流压缩结果：结论、依据、风险、建议、改动文件和验证结果。

每条证据面只设一个 owner。Main 不重复调查同一证据面，否则所谓隔离只会变成双倍成本。

## Work Package 不等于派发节点

Work Package 定义的是目标、上下文、恢复和验收边界。它可以由 Main 完成，也可以在运行时交给 Child。

`推荐 Agent` 只有在已经决定派发后才用于选角色；它不强制创建 Child。平台不支持推荐角色时，可以让普通执行 Agent 读取对应领域指引，不因此阻塞 Case。

## 当前角色与领域指引

AIPD 当前包含：

- `aipd_adoc_retriever`：大量项目认知或多条独立认知线的检索与压缩。
- `aipd_product_manager`：Design requirements 的目标、规则、confirmed / assumed / open 梳理。
- `aipd_vue_architect`：Vue 页面、组件、交互、状态与整体架构。
- `aipd_vue_provider`：Vue `useXxx`、provide / inject、API 数据源和字段对齐。

角色指引是领域边界，不是自治权限。派发不会扩大 Case 范围，也不会授予安装、远端写入、删除等外部副作用权限。

## 平台上下文继承

具体平台怎样 fork 或传递上下文属于平台实现，不写成 AIPD 的通用承诺。通用原则只有两个：

1. 已落盘的 Case / Work Package / `_adoc` 足以恢复时，优先传最小文件上下文。
2. 任务强依赖尚未落盘的主线判断时，才增加会话上下文继承。
