# 分身 Agent

AIPD 的分身 Agent 模型不是“主 Agent 派工给低上下文子 Agent”。

更准确的模型是：分身 Agent 是从主 Agent 当前认知 fork 出来的克隆体。它继承同一份项目认知、对话背景、任务方向和判断逻辑，然后进入局部探索或执行分支。

## 主 Agent 做什么

主 Agent 保留主线：

- 用户沟通。
- 意图判断。
- 目标边界。
- 分身调度。
- 方案审核。
- 最终判断。
- 状态写回。

主 Agent 不应该吞下所有扫描、验证、日志、diff 和批量修改过程成本。

## 分身 Agent 做什么

分身 Agent 负责进入局部分支，完成当前目标。

它可以读取文档、扫描代码、核对接口、运行验证、整理 diff、修改文件。完成后只回流压缩结果：

- 结论
- 依据
- 风险
- 建议
- 改动文件
- 验证结果

分身 Agent 不主动继续分身，不擅自扩大范围，不推进下一步。

## Case Execute 和分身

Case Execute 只是把分身探索固化为 work package。

每个 work package 都有明确文件入口、上下文索引、横向模块、验收标准和不做范围。执行 Agent 读取这些文件后完成局部任务，再把结果回流给主线。

普通 AIPD 对话、轻量修改、调研、验证和正式 case 都可以使用同一套分身逻辑。

## fork_context

在 Codex 语境中，AIPD 默认创建分身 Agent 时使用 `fork_context: true`。

这表示分身继承主 Agent 当前认知状态，而不是从空白上下文开始。继承之后，分身仍应以 case / phase / work package / `_adoc/` 文件为事实源，避免依赖不可恢复的聊天记忆。
