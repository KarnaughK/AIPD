# Inbox

Inbox 是 AIPD 的临时收件箱。

它用于接住暂时还没整理清楚的信息，例如灵感、资料线索、后续想法、用户临时反馈或还不确定是否值得建 case 的事项。

## 和三条主线的关系

Inbox 不单独成为一整层。

它是辅助能力，可能流向不同位置：

- 变成稳定项目认知：后续 weave 进 L1-L5、map 或局部 README。
- 变成执行事项：后续创建 case，再拆 step。
- 保持临时记录：暂时没有足够价值或边界，不进入长期知识库。
- 被丢弃：确认不再有用后，不污染 `_adoc`。

## 适合放入 Inbox 的内容

- “先记一下，之后再整理”的想法。
- 一个还没确认归属的产品点。
- 一条外部资料或竞品线索。
- 一个可能形成 case 的问题。
- 一段暂时不值得写进 L3 / L4 / L5 的经验。

## 不适合放入 Inbox 的内容

- 已经确认要执行的具体事项：应创建 case。
- 已经稳定的项目规则：应 weave 到 L3 / L4 / L5 / README / map。
- 执行过程和验收记录：应留在 case / step。

Inbox 的价值是降低记录门槛，但它不是长期知识库。进入 Inbox 只是暂存，后续仍要判断归属。
