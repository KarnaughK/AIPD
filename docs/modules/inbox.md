# Inbox

Inbox 是低承诺度的临时收件箱。用户明确说“先记一下、先存一下、放 Inbox、回头整理”时，用 `/aipd-inbox` 接住信息，但不提前判断它已经是任务或长期事实。

## 为什么需要临时层

如果每条灵感都立刻创建 Case，会制造大量未确认目标；如果每条经验都立刻写进 L3 / L4 / L5，会污染知识库。

Inbox 允许先记录，再在有上下文时决定去向。

## 适合放什么

- 一条资料或竞品线索。
- 还没确认价值和边界的产品想法。
- 用户临时反馈或观察。
- 可能形成 Case、SOP 或长期认知，但现在无法判断的材料。

## 不适合放什么

- 已经明确要执行并有验收标准的事项：创建 Case。
- 已经成为稳定项目规则的信息：用 Weave 写回正确层级。
- 当前 Case 的调研、设计和执行状态：留在对应 phase / Work Package。
- 待本人完成的具体待办：使用真正的任务管理系统，不把 Inbox 当 todo。

## 后续去向

```text
Inbox 条目
├── 值得执行且边界清楚 -> Case
├── 已经稳定           -> Weave 到 L3 / L4 / L5 / README / map
├── 可重复项目动作     -> SOP 候选
├── 暂时不判断         -> 继续保留
└── 已无价值           -> 清理
```

Inbox 的价值是降低捕获成本，不是把分类工作永远往后拖。整理时仍要检查知识 owner、执行边界和是否真的稳定。
