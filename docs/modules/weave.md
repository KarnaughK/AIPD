# Weave

Weave 是 AIPD 的反向编织能力。

它负责把讨论、开发、work package 结果、case 归档、代码 diff、错误日志和外部资料中产生的新信息，判断并回写到合适位置。

## Weave 不是什么

Weave 不是聊天存档。

它不要求把每次对话、每段日志、每个临时猜测都写进项目知识库。这样会污染 `_adoc/`，也会让 Agent 更难检索。

Weave 关注的是稳定信息。

## 常见回写方向

| 信息类型 | 回写位置 |
|---|---|
| 新核心概念、别名、误解 | L3 |
| 新产品功能边界、业务规则 | L4 |
| 新实现逻辑、跨模块规则、调试经验 | L5 |
| 页面、弹窗、组件内部入口 | 局部 README |
| 高频检索路径 | `_adoc/map.md` 或细节 map |
| 一次性执行过程 | case / work package |

## Weave Candidate

执行型任务不一定直接修改长期 ADOC。

更稳妥的做法是先产出 Weave Candidate：说明这次任务产生了哪些可能值得回写的稳定信息，建议写到哪里。

之后由 `aipd-weave` 统一判断知识归属、索引更新和旧知识冲突。

## 与 case 归档的关系

case 归档时通常会整理本次事项的结果、风险、经验和 Weave Candidate。

归档不是把 case 删除，而是把一次任务的执行过程收束，让稳定知识进入长期项目记忆，让一次性过程留在 case 记录里。
