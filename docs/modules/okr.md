# OKR

OKR 是 AIPD2 的长期目标管理辅助能力。

它不替代 `_adoc` 知识库，也不替代 Case / Step。它回答的是：当前阶段为什么要做这些事项，以及哪些结果说明项目方向被推进了。

## 和三条主线的关系

OKR 主要服务第二层：Case / Step 开发逻辑。

它可以帮助判断：

- 哪些 case 更接近当前阶段目标。
- 某个 step 是否还在推进目标，而不是变成局部忙碌。
- case 归档时，哪些经验值得进入长期知识库。

OKR 也会连接 L1 Intent，因为阶段目标不能偏离项目长期方向。

## OKR 不是什么

OKR 不是 todo 列表。

它不负责记录每个具体任务的执行细节。具体事项应该进入 case，具体执行应该进入 step。

OKR 也不是产品路线图的完整替代品。它只保留当前阶段对 AI 协作有用的目标、关键结果和判断标准。

## 常见使用方式

一个常见链路是：

```text
L1 Intent
-> OKR
-> Case
-> Step
-> Weave Candidate
```

L1 定方向，OKR 定阶段目标，Case 固化一次事项，Step 执行局部任务，Weave Candidate 再判断执行经验是否回到知识库。
