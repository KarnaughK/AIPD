# 05 weave 项目记忆回写

项目知识库如果只读不写，很快会过期。

AI 协作中经常产生新信息：

- 讨论中确认了一个产品边界。
- 一个 step 执行后发现了稳定实现规则。
- 一次 bug 修复暴露出某个接口约定。
- 一个 case 归档时总结了可复用经验。
- 外部资料改变了对场景或竞品的理解。

如果这些信息只留在聊天里，下一次 Agent 仍然继承不到。

## Weave 做什么

Weave 负责把稳定信息编织回项目认知。

它不是保存所有聊天，也不是把每个过程都写进 `_adoc/`。它要判断哪些信息已经稳定到值得成为项目记忆，并放到合适位置：

- 新核心概念、别名、常见误解：回写 L3。
- 新产品功能边界、业务规则：回写 L4。
- 新实现逻辑、跨模块规则、调试经验：回写 L5。
- 页面、弹窗、组件内部入口：回写局部 README。
- 高频检索路径：回写 `_adoc/map.md` 或细节 map。
- 一次性执行过程和验收记录：留在 case / step。

## Weave 和 learn

AIPD2 里还有 `aipd2-learn`。

两者分工不同：

- `aipd2-weave` 面向当前项目，把稳定信息回写到当前项目的 `_adoc/`、map、局部 README 或 case。
- `aipd2-learn` 面向 AIPD2 框架自迭代，采集会话定位信息，帮助之后诊断 skill 和规则是否需要改进。

## 什么时候需要 weave

不是每次改文件都需要 weave。

值得 weave 的信息通常满足至少一个条件：

- 会影响后续多个任务。
- 会改变项目核心概念、产品边界或工程规则。
- 能减少 Agent 下次检索成本。
- 暴露了旧文档和当前事实的冲突。
- 是一个稳定经验，而不是一次性过程细节。
