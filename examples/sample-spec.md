# Product Spec: 官方文档翻译

> **核心能力**：将 OpenClaw 官方英文文档翻译成多语言版本，解决非英语用户的"怎么用"问题

## 1. 核心概念

### 1.1 文档同步

- **定义**：跟踪 https://docs.molt.bot/ 的内容更新，保持本地文档与官方同步
- **价值**：确保用户获取的是最新、准确的产品信息

### 1.2 多语言翻译

- **定义**：将英文文档翻译成中文、日文等多语言版本
- **价值**：降低非英语用户的学习门槛，扩大受众范围

### 1.3 文档结构映射

- **定义**：保持与官方文档相同的章节结构和导航层级
- **价值**：用户可以在官方文档和本站之间无缝切换，降低认知负担

## 2. 文档组织

### 2.1 章节分类

根据官方文档结构，主要包含：

- **Start Here**：入门指南（Getting started, Setup, Pairing 等）
- **Install & Updates**：安装和更新
- **CLI**：命令行工具
- **Core Concepts**：核心概念
- **Gateway & Ops**：网关和运维
- **Channels**：频道集成
- **Tools & Skills**：工具和技能
- **其他高级主题**

### 2.2 优先级

**P0（MVP 必须）**：
- Start Here（入门指南）
- Install & Updates（安装和更新）
- Core Concepts（核心概念）

**P1（后续补充）**：
- CLI 命令详解
- Gateway & Ops
- Channels

**P2（长尾内容）**：
- 高级配置
- 平台特定指南

## 3. 关键用户旅程

### Journey: 新用户查看入门指南

1. 用户访问 OpenClaw.dog
2. 系统检测浏览器语言，自动跳转到对应语言版本（如 /zh/）
3. 用户点击"入门指南"
4. 系统展示翻译后的 Getting Started 文档
5. 用户可以通过语言切换器切换到其他语言版本

### Journey: 用户搜索特定功能

1. 用户在搜索框输入关键词（如"WhatsApp"）
2. 系统在当前语言的文档中搜索
3. 系统展示匹配的文档列表
4. 用户点击进入具体文档页面

### Journey: 用户查看文档更新

1. 官方文档更新
2. AI Agent 检测到更新
3. AI Agent 翻译新内容
4. 系统自动构建并部署
5. 用户访问时看到最新内容

## 4. 验收标准

- [ ] 完成 Start Here 章节的中文翻译（至少 5 篇核心文档）
- [ ] 文档页面可以正常渲染 Markdown 内容（标题、列表、代码块、链接）
- [ ] 支持中英文切换（语言切换器）
- [ ] 文档导航结构与官方保持一致
- [ ] 移动端适配（响应式布局）
- [ ] 代码块支持语法高亮
- [ ] 页面加载速度 < 2 秒

## 5. 依赖关系

- **依赖**：无（这是第一个实现的模块）
- **被依赖**：
  - 第三方教程收集（需要复用文档展示组件）
  - 最佳实践案例（需要复用 Markdown 渲染逻辑）
  - 内容分发（需要复用多语言路由）
