# Think：Case 编号是否还需要版本语义

## 问题

Case 的旧编号形如 `c0.14`，外观接近版本号，但 Case 实际是一个短周期目标容器。高频开发中一天可能推进多个事项，继续区分大版本、小版本和修订版本没有稳定含义。

## 判断

- 产品版本负责表达对外发布和兼容性，不应由 Case ID 代替。
- Case 内的小修小补可以通过 phase 回跳或新增 Work Package 完成，不需要第三段编号。
- 框架更换、商业能力重建等大变化是否新开项目，应依据产品和代码边界决定，而不是通过 Case 大版本表达。
- Case 只需要一个项目内单调递增的身份号。

## 结论

- Case 使用 `cN-slug`。
- Work Package 使用 `wp-NN-slug`。
- Case 外引用 Work Package 时使用 `cN/wp-NN`。
