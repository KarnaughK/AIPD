## 目标

把面向人的学习文档调整为“三条主线 + 辅助能力”的新版教学结构，并保留旧版问题引入和 modules 能力字典。

## 已确认设计输入

- 新版教学文档不再只按功能出现顺序讲，而是先讲三条主线：知识库最小闭环、Case / Step 开发逻辑、AI 原生代码架构实验。
- 第一层知识库最小闭环包含知识库本体、存和取：L1-L5 是知识库本体，L6 是被索引的真实代码；Weave / 反向编织负责维护；map 负责读取。
- 第二层 Case / Step 解决大任务上下文不够、状态容易散的问题；OKR、Case、Step、分身 Agent、执行记录、归档和 Weave Candidate 都服务这一条执行线。
- 第三层 AI 原生代码架构实验讨论代码组织范式：纵向黑箱、降低牵连面、Decouple first, DRY later。它是探索性主线，不应写成已经定型的行业共识。
- OKR、Inbox、MMD / Mermaid 等能力不单独升成一整层。OKR 偏第二层长期执行管理；Inbox 是临时信息捕获入口；MMD / Mermaid 按用途归属到知识库、执行拆解或代码架构实验。
- 旧版从 Vibe Coding 到 Agent Coding 的问题引入和 `docs/modules/` 能力字典应保留。新版改的是教学结构，不是推翻旧文档材料。
