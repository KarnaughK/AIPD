# Design Decision Log

## D1：用记忆循环取代“三条主线”作为首次叙事

- **决定**：README 和 guide 以任务前读认知、任务中文件承接、任务后回写经验为主线。
- **原因**：它是用户结果导向的连续故事，能够自然容纳 ADOC、map、Case、Weave 和 Agent 调度；三条主线继续作为进阶架构理解。

## D2：保留现有文件路径

- **决定**：不重命名六篇 guide 和 modules 文件，只重写标题、职责和内容。
- **原因**：保护现有 GitHub 深链接；用户可见导航由 `docs/README.md` 提供，文件名历史不应换来链接破坏。

## D3：不新增独立文档站

- **决定**：本轮用 Markdown 信息架构、真实流程和链接完成产品化，不引入 VitePress / Docusaurus 等站点工程。
- **原因**：当前问题是叙事与内容职责，不是渲染技术；站点会扩大维护面且不直接解决首次学习断点。
