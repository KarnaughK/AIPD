# SOP 地图

本文件是 `_adoc/sop/` 的局部 map。它负责让 Agent 按用户说法、项目动作和运行场景找到可复用 SOP。

SOP 的存储可以按实际项目情况调整，不强行塞进 L4 或 L5。索引要求更硬：只要一个 SOP 需要被复用，就应能从 `_adoc/map.md` 或本文件找到。

## 使用规则

- 总入口仍是 `_adoc/map.md`；本文件负责 SOP 目录内部的细分索引。
- 高频 SOP 应同时在 `_adoc/map.md` 暴露一跳入口。
- 普通 SOP 可以先只登记在本文件。
- SOP 可以跨多个前端、后端、脚本、数据源或外部工具；不要因为它调用了某类代码就把它归入 L5。
- 如果某个 SOP 还只是草稿，可以先建目录收纳材料，后续再补完整执行说明。

## 存储约定

当前只采用最小结构：

```text
_adoc/sop/{sop-name}/
└── README.md
```

`README.md` 可先记录：

- 目标：这个 SOP 反复执行什么动作。
- 输入：每次运行需要用户或系统提供什么。
- 步骤：Agent 按什么顺序读上下文、调用工具、执行代码、分析和输出。
- 工具：涉及哪些脚本、API、外部服务、数据源或代码入口。
- 输出：执行后要交付什么。
- 收尾：是否生成 Case 候选、Weave Candidate 或更新 map。

后续如果某个 SOP 变复杂，可以在该目录下继续增加脚本、样例、执行记录或子文档。
