# AIPD Project Entry

本项目使用 AIPD 维护项目认知。

AIPD 是面向 AI 协作的软件项目认知框架。它把项目从方向、调研、核心模型、产品结构到研发实现的关键判断沉淀下来，让 AI 不只看到代码结果，也能理解为什么这样设计。

## 认知层级

- **L1 Intent**：项目方向、目标和长期取舍。
- **L2 Research**：用户、场景、需求、竞品和调研结论。
- **L3 Core**：核心对象、核心流程、数据模型和系统成立方式。
- **L4 Product**：产品模块、功能边界、业务规则和交互行为。
- **L5 Dev**：研发实现、工程约束、调试经验和维护规则。

任务处在哪个层级，就读取对应层级及必要上下游认知。

## 文档位置

- `_adoc/index.md` 是项目认知入口。
- `_adoc/L1-*` 到 `_adoc/L5-*` 存放项目级认知。
- 代码模块、页面、组件的局部认知，优先放在模块根目录或页面目录的 `README.md`。
- `_adoc/L5-dev/` 更适合放跨模块研发规则、工程约束、索引和通用 SOP。

## 执行概念

- **Case**：一次具体事项，记录目标、上下文索引、边界、steps 和归档信息。
- **Step**：Case 内可派发给子 Agent 的最小执行单元。
- **OKR**：阶段目标，用来判断工作是否推进了项目方向。

## Main / Sub Agent Mode

AIPD 默认保护 Main Agent 上下文。

**Codex 默认策略：创建子 Agent 时使用 `fork_context: true`。**

Main Agent：

- 负责用户沟通、意图判断、任务边界确认、子 Agent 调度、方案审核和结果汇总。
- 尽量不直接承担探索、验证、批量修改、执行等会污染上下文的工作。

Sub Agent：

- 由 Main Agent fork 出来，并继承当前对话上下文。
- 只执行 Main Agent 派发的当前任务。
- 默认先给方案，不直接执行；等待 Main Agent 确认后再执行。
- 不擅自扩大范围，不主动推进下一步。
- 返回简洁结论、依据、风险和建议，完成后停止。

派发规则：

- 创建 Codex 子 Agent 时，默认使用 `fork_context: true`。
- 每个 Step 默认创建一个新的子 Agent。
- 如果当前消息明确声明“你是 AIPD 子 Agent”，进入 Sub Agent 模式。
- 否则默认 Main Agent 模式。

## 冲突规则

用户当前指令定义本次任务目标。AIPD 提供项目默认认知和历史判断。

当用户指令与 AIPD 认知冲突时，先指出冲突和风险，再继续。
