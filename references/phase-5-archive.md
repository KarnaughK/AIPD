# 阶段 5: 归档计划

**目标**: 计划完成后归档，更新索引，准备下一轮迭代

**触发条件**: Plan 中所有任务都标记为完成

## 执行步骤

### 步骤 1: 确认所有任务完成

```
【计划完成确认】

v0.2-search 的所有任务已完成:
✅ 创建搜索 API 端点
✅ 实现基础查询逻辑
✅ 添加搜索结果分页
✅ 实现高亮显示
✅ 编写测试用例

接下来进行文档同步检查...
```

### 步骤 2: 文档同步检查（归档前必须完成）

检查 Plan 中的「文档同步」章节，逐项确认：

```
【文档同步检查】

📁 Tech 文档:
- [ ] tech/index.md - 是否需要更新？
- [ ] 涉及的项目文档 - astro/README.md / web/README.md 等
- [ ] 是否有新增技术模块需要创建文档？

📁 Business 文档:
- [ ] 涉及的业务模块 README.md 是否需要更新？
- [ ] 是否有新增 SOP 需要记录？

📁 大索引:
- [ ] _adoc/index.md 是否需要更新？

---
请确认以上文档是否都已同步，或标记为"无需更新"。
```

### 步骤 3: 引导用户完成文档更新

- 如果有文档需要更新 → 先完成更新
- 如果用户确认无需更新 → 继续归档流程
- **不允许跳过此步骤**

### 步骤 4: 更新 Plan 索引

询问用户：这个 Plan 属于哪个模块？
- 可选：Homepage / Docs / i18n / 翻译系统 / SEO / 基础设施 / 其他

然后更新 `_adoc/plan/index.md`：
```bash
# 在对应模块下添加记录
# 格式：- [vX.Y-xxx](archive/vX.Y-xxx.md) - {一句话描述}
```

同时将该 Plan 从「当前进行中」部分移除（如果有的话）。

### 步骤 5: 移动文件到 archive/

```bash
mv _adoc/plan/v0.2-search.md _adoc/plan/archive/v0.2-search.md
```

### 步骤 6: 告知下一步

```
✅ 计划已归档

文件: _adoc/plan/archive/v0.2-search.md
索引: 已更新 _adoc/plan/index.md（归类到 {模块名}）

文档同步状态:
✅ tech/ 已更新
✅ business/ 无需更新
✅ index.md 无需更新

---
下次调用 /adoc 将引导创建新的迭代计划。
```

---

## 关键原则

- **文档同步是归档的前置条件**，不能跳过
- **更新索引是归档的必要步骤**，确保历史可追溯
- 确保代码变更都反映在相关文档中
- 保持文档与代码的一致性
