# 沉淀指引

## 触发时机

- Plan 归档完成后，用户确认需要沉淀
- 对话中产生了重要决策/设计，用户主动要求沉淀

## 执行方式

交给子 Agent 执行（上下文消耗大，不污染主 Agent）。

## 子 Agent 任务

### 输入

- Plan 文件路径（或对话摘要）
- 项目 `_adoc/` 根目录路径

### 执行步骤

1. 读 `_adoc/index.md`，了解现有文档结构
2. 读 plan.md（或对话摘要），了解这次做了什么
3. 读相关代码（plan 中涉及的文件）
4. 判断哪些 `_adoc/` 文档需要更新（新建 or 追加）
5. 更新文档 + 更新对应模块的 index.md

### 约束

- 只更新与本次 plan/对话直接相关的文档
- 不重写整个文档，优先追加或局部修改
- 用 git 版本控制兜底，直接更新不需要提前确认

### 返回

```
沉淀完成：
- 更新了 X 个文档
- 新建了 X 个文档
```
