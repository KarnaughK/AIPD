# AIPD 未来演进计划

## 已完成

### 多 Agent 调度（v1 基础版）- 2026-03-22

将 Team Agent Skill 的多 Agent 调度能力融合进 AIPD：

- **版本号升级为 A.B.C 三级**：B 级功能版本 + C 级迭代步骤
- **Plan 支持多种类型**：dev / research / review
- **阶段 4 重写为调度模式**：主 Agent 不再自己执行，派子 Agent 干活
- **子 Agent 指南**：worker（开发）+ researcher（调研）
- **步骤文件模板**：C 级步骤的标准格式
- **降级模式**：简单任务可以退回主 Agent 直接执行

**设计决策**：
- 不是为了并发加速，而是为了主 Agent 上下文干净 + 随时响应用户
- 子 Agent 串行执行（后一步依赖前一步），不做并行
- 步骤文件自包含（列出需要读的上下文文档），子 Agent 不需要知道 Plan 全貌

## 待做

### SOP 资产库

- 每个流程跑通一次就沉淀成 SOP
- SOP 内置到 AIPD 框架中，开箱即用
- 每个 SOP 可以直接派子 Agent 去执行

### 并发执行（进阶）

当前是串行执行（v0.1.1 → v0.1.2 → ...）。未来如果步骤之间确实独立（不同模块），可以支持并行：
- Plan 中标注可并行的步骤组
- 主 Agent 同时派多个子 Agent
- 需要解决文件冲突问题（git worktree 等）

### Agent SDK 集成

- 不用内置 subagent，而是用 Claude Code 的 Agent SDK 调新的 Claude Code 实例
- 每个实例能读 `_adoc/` 文档体系，拿到自己的步骤独立干活
- 好处：更强的隔离性、更大的上下文窗口

### 渐进式文档加载优化

- 之前所有内容写在 skill.md 里，现在拆到 references/ 下按阶段加载
- 后续继续往下叠加更多阶段文件和 SOP 文件

## 待解决的问题

1. **并发冲突**：多个 Agent 同时改代码时的文件冲突处理
2. **依赖分析**：plan 中任务的依赖关系自动判断
3. **步骤粒度自动化**：让 AI 更好地判断步骤该多大
