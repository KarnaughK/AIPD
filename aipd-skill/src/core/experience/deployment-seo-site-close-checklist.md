# 部署型 SEO 网站 Close 检查实践经验

## 来源

- 来源类型：已完成外部项目 case 的上线部署复盘。
- 经验性质：实践经验库条目。用于提醒 Agent 在部署型 case 的 Verify / Close 阶段补齐基础上线检查；不是通用 core 规则，也不是 SEO 效果检测工具。

## 适用场景

在 AIPD Case Verify / Close 阶段，如果本 case 涉及上线部署，并且目标是 SEO 网站、内容站、工具站、游戏站或其他搜索流量承接页面，应读取本经验。

如果只是内部后台、纯 API、离线脚本或不对搜索引擎开放的功能，不需要套用本清单。若项目已有专门 SEO 检测工具或 SOP，应优先使用项目自己的工具 / SOP，本经验只补基础部署验收视角。

## 问题背景

部署型 case 很容易只检查“代码能 build / 页面能打开 / API 能返回”，然后直接 Close。但对 SEO 网站来说，上线是否真的完成，还取决于预发环境、正式域名、运行时配置、跳转行为、旧平台是否仍在服务、sitemap / robots / canonical 等基础入口是否稳定。

这些检查不应该写成 AIPD core 强规则，因为它们只在部署上线场景成立；但作为实践经验，它能帮助 Agent 在 Verify / Close 时避免漏掉常见上线风险。

## Verify / Close 基础检查

### 环境与部署目标

- 明确预发 / PE / staging 环境的 URL 和正式环境 URL。
- 明确部署平台、项目名、runtime preset、构建产物目录和运行时兼容配置。
- 确认正式项目和预发项目不是误用同一个环境。
- 如果存在旧平台，记录切换前后如何判断当前请求由哪个平台承载，例如响应头、平台域名、部署 ID 或控制台状态。

### 运行时配置

- 检查生产 runtime 需要的 env、secret、数据库 binding、对象存储 binding、第三方 API key 是否已经配置。
- 写入 secret 后，如果平台要求重新部署或重新发布，应重新部署后再验收。
- 检查 active model / feature flag / environment name 等会改变业务结果的关键配置。
- 确认本地调试 adapter、CLI fallback、mock 或临时数据源没有进入正式请求链路。

### 页面与域名

- 预发 URL 返回预期 HTTP status，根页面和主要语言 / 主要落地页可访问。
- 正式域名绑定后，裸域、`www`、主要语言路径和主要落地页可访问。
- `www` 到裸域或裸域到 `www` 的跳转符合项目约定。
- 旧平台不再响应正式域名，或旧平台只保留明确的回滚 / 备用路径。
- 如果项目有浏览器语言检测或历史语言 cookie，确认它不会自动改写 SEO URL。

### API 与动态主流程

- 核心 API 在预发和正式环境都能 smoke test。
- 首次请求和缓存命中请求都要测；只测缓存命中容易掩盖第三方 API、写库或对象存储问题。
- 记录关键 API 的粗略耗时，确认没有明显不可接受的冷启动、外部 API 或数据库延迟。
- 对当天数据、seed 数据或发布数据有依赖的接口，应区分“部署失败”和“缺少业务数据”。

### SEO 基础入口

- 主要页面的 `title`、`meta description`、canonical、hreflang / alternate、robots、sitemap 入口符合项目预期。
- sitemap 如果跳转到 sitemap index，应确认最终 XML 可访问。
- 对多语言站点，主要语言路径应保持路径语义稳定，不应被浏览器语言、cookie 或地理位置自动 302 到另一个语言页面。
- 只做基础入口检查，不在这里做完整 SEO 效果评估或排名评估；这些应交给专门工具或项目 SOP。

### 回滚与残留风险

- 记录切换域名 / DNS / custom domain 后的回滚路径。
- 记录只能从当前网络视角验证的限制，例如 DNS 全球传播、地区边缘节点状态、搜索引擎重新抓取延迟。
- Close 时把未解决但不阻塞上线的风险写入残留风险或后续观察项。

## Close 输出建议

部署型 SEO 网站 case Close 时，至少说明：

- 预发环境和正式环境是否都验收过。
- 正式域名、主要路径、`www` 跳转和旧平台退出是否确认。
- 核心 API / 动态主流程是否在正式 runtime 跑通。
- SEO 基础入口是否做过最小检查。
- 哪些风险需要后续观察，哪些后续体验或 SEO 优化应进入新的 case。

## 不做

- 不评估 SEO 排名效果。
- 不替代专门 SEO 检测工具。
- 不要求所有部署 case 都执行本清单。
- 不把具体业务项目的域名、项目名、secret 名或本机路径写入 AIPD 通用经验。
